# Luxwap Flutter 集成测试用例

## 0. 测试范围

这些用例用于真实端到端 UI 自动化测试，目标是运行中的 Flutter Windows 客户端和真实 RuoYi 后端 `http://127.0.0.1:8081`。

测试不 mock 后端接口。需要验证 UI 状态、后端副作用、本地持久化、xray 运行行为和用户可见提示。

## 1. 环境前置条件

- RuoYi 后端运行在 `127.0.0.1:8081`。
- MySQL 已初始化有效用户、套餐、线路和线路配置。
- Flutter Windows 客户端能访问内置的 `bin/xray/xray.exe`。
- Google/X OAuth 测试前，本地 `3000` 回调端口未被占用。
- 测试机器能访问 OAuth 提供商网站。
- 至少存在一条可用 VLESS 线路，并且远端服务端已同步测试用户 UUID。
- 已存在测试账号：
  - 邮箱：`lilibestcoder@163.com`
  - 密码：`123456`
- 注册、找回密码、换邮箱测试需要一个一次性测试邮箱，或后端提供验证码测试旁路。

## 18. 推荐自动化实现顺序

1. 添加 `integration_test` 依赖和 Windows 集成测试入口。
2. 增加测试前置清理逻辑：清 token、清本地配置、关闭残留 xray。
3. 增加后端健康检查和测试数据检查。
4. 先实现登录、退出、主菜单导航测试。
5. 再实现线路加载、线路选择、线路持久化测试。
6. 再实现设置持久化和 xray 配置断言。
7. 再实现 xray 启停和测速用例。
8. 最后实现 OAuth 测试，因为它依赖外部浏览器和三方授权。
